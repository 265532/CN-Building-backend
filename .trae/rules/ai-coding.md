# AI Agent 专属编码与协作规约

作为本项目的 AI 编程助手，在执行任何代码生成、修改或架构设计任务时，必须**强制遵守**以下基于 `.trae/rules/ai-coding.md` 提取的 AI 专属规约。

## 1. 架构与设计原则

- **分层架构**：强制遵循 `Controller -> Service -> Repository` 结构。生成代码时，绝不允许在 Controller（表示层）中编写复杂业务逻辑，Controller 仅保留参数校验和依赖注入调用。
- **接口规范**：生成的 API 路径必须是 RESTful 风格（复数名词，小写短横线，如 `/api/v1/users`）。API 返回体必须强制封装为 `{"code": 200, "message": "success", "data": ...}` 格式。

## 2. 命名与文件结构

- **文件与类命名**：生成新文件时，文件名强制使用 `snake_case`（如 `user_service.py` / `user_service.js`），类名强制使用 `PascalCase`（如 `UserService`）。
- **目录放置**：准确将代码放入对应目录：`api/` (路由/接口), `core/` (配置), `models/` (数据模型), `services/` (业务逻辑), `utils/` (工具), `tests/` (测试)。

## 3. 代码质量与依赖

- **类型与签名**：生成的所有函数/方法必须包含完整的类型提示（Type Hints）和明确的函数签名。
- **导入规范**：强制使用绝对路径导入（如 `import { UserService } from '@/services/user_service'`），禁止使用相对路径。
- **依赖管理**：如需新增依赖，必须指定明确的固定版本号（如 `package==1.2.3` 或 `package@1.2.3`），绝对禁止使用范围符号（如 `^`, `>=`, `~`）。

## 4. 数据库与持久化

- **模型字段**：生成的数据库模型必须包含公共字段：`id`, `created_at`, `updated_at`。表名强制使用复数。
- **禁止直连修改**：需要修改数据库结构时，必须生成对应的**数据库迁移脚本**（Migration Script），绝对禁止提示用户直接操作数据库或尝试执行 DDL。
- **分页与性能**：生成的列表查询接口，必须默认实现分页参数（`limit` / `offset`）。

## 5. 安全与日志

- **密钥管理**：严禁在代码中硬编码任何密码、Token、API Key 或数据库连接串。必须使用环境变量（如 `process.env.DB_URL` 或 `os.getenv('DB_URL')`）。
- **日志规范**：生成的日志记录代码必须包含关键上下文信息，且必须对敏感信息（密码、手机号等）进行脱敏处理。

## 6. 任务交付与文档

- **强制交接文档**：在完成任何重要模块或复杂任务后，AI 必须主动生成或更新交接文档至 `doc/<module>/handover.md`。内容需包含：已实现功能、Todo 事项、技术决策说明、后续建议。
- **冲突处理准则**：当发现项目文档说明与实际代码实现存在冲突时，**必须以实际生效的代码为准**，并在最终回答中向用户明确指明冲突的依据和你的决策逻辑。
